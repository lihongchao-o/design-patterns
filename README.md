# design-patterns
本项目设计模式的应用举例。           
设计模式是例如像C++这种面相对象的设计语言对反复出现的问题的可重用解决方案。
创建型设计模式：抽象实例化过程；
结构型设计模式：类和对象如何组合以形成更大的结构；
行为型设计模式：对象之间的职责分配。

#### 创建型模式
- [Abstract Factory]：产品对象集合
- [Builder]：创建一个复合对象
- [Factory Method]：被实例化的对象的子类
- [Prototype]：被实例化的对象类
- [Singleton]：一个类的唯一实例
#### 结构型模式
- [Adapter]：一个对象的接口
- [Bridge]：对象的实现
- [Composite]：对象的结构和组成
- [Decorator]：没有子类化的对象的职责
- [Façade]：子系统的接口
- [Flyweight]：对象的存储成本，享元
- [Proxy]：如何访问对象(它的位置)
#### 行为型模式
- [Chain of Responsibility]：能够满足请求的对象
- [Command]：何时以及如何完成请求
- [Interpreter]：语言的语法和口译
- [Iterator]：如何访问聚合的元素
- [Mediator]：对象如何和哪些对象相互作用
- [Memento]：什么私人信息存储在对象之外，以及何时存储
- [Observer]：依赖对象如何保持最新
- [State]：对象的状态
- [Strategy]：算法
- [Template Method]：算法的步骤
- [Visitor]：可以应用到对象的操作，而不需要改变它们的类
